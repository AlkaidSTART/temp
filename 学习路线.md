## 二、 算法学习路线（零基础 3 阶段）

### 第一阶段：夯实数据结构基础（前 2 周）
**不要直接刷题！** 先用 JavaScript/TypeScript 把以下结构自己实现一遍：
* **数组/字符串：** 掌握常用 API（`slice`, `splice`, `reduce`, `map`）。
* **链表：** 理解指针、反转链表、检测环。
* **栈/队列：** 理解先进后出/先进先出（前端常考：有效括号、滑动窗口）。
* **树：** 重点掌握**二叉树的前中后序遍历**（递归与非递归）。

### 第二阶段：LeetCode 必刷分类题单（第 3-8 周）
针对前端面试，不需要刷完 3000 题，集中攻克以下 **80-100 题** 即可。

| 分类 | 核心重点 | LeetCode 经典题号 |
| :--- | :--- | :--- |
| **双指针** | 解决有序数组查找 | 1, 15, 26, 88 |
| **滑动窗口** | 字符串子串问题 | 3, 76, 209 |
| **二叉树** | 深度/广度优先搜索 | 102, 104, 226, 144 |
| **动态规划** | 爬楼梯、背包问题（最难，先做简单的） | 70, 121, 322 |
| **链表** | 基本操作 | 21, 141, 160, 206 |
| **排序与搜索** | 快速排序、二分查找 | 704, 215, 912 |
| **栈与队列** | 逻辑匹配 | 20, 232, 155 |

### 第三阶段：前端“特供”算法（冲刺期）
大厂前端面试不仅考 LeetCode 原题，更喜欢考 **JS 手写实现**：
1.  **Promise.all / Promise.race** 的算法实现。
2.  **树形结构转换：** 扁平数组转 Tree（常见于菜单导航）。
3.  **深拷贝 (Deep Clone)：** 考虑循环引用。
4.  **防抖 (Debounce) 与 节流 (Throttle)**。
5.  **虚拟 DOM 转真实 DOM** 的递归算法。
